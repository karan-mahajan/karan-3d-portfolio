## What it does (code-level)
1. Gets/creates collection `map`, links to scene root.
2. Creates 4 objects:
   - `cameraTerrain` (CAMERA, references `Camera`) at `(0, 0, 55.74)` rot 0 — top-down ortho camera for the minimap.
   - `night` (LIGHT, references `Area`) at `(17.53, -38.46, 105.82)` rot `(-0.668, 0.004, -4.378)` scale 245.91 — large area light for night-time minimap rendering. `hide_viewport=True, hide_render=True, hide_select=True`.
   - `day` (LIGHT, references `Area.001`) at the SAME position, rotation, and scale — same area light but for day rendering. NOT hidden.
   - `cameraVehicle` (CAMERA, references `Camera.001`) at `(0, 0, 55.74)` — vehicle minimap variant, also hidden (`hide_viewport=True, hide_render=True, hide_select=True`).

## Technique / recipe
- **Two cameras (terrain + vehicle)** — the runtime can render the minimap in two modes:
  - `cameraTerrain` = top-down ortho of the static world.
  - `cameraVehicle` = top-down centered on the player vehicle (presumably the runtime moves this camera with the car).
- **Two lights (day + night)** — same scale/rotation, but different brightness/color via the underlying light datablock (`Area` vs `Area.001`). Runtime toggles which is active based on time-of-day.
- **`hide_viewport/hide_render/hide_select=True`** on three of the four — keeps these objects out of the main scene viewport during authoring. They're still in the .blend; just invisible during Blender editing.
- **The view-layer exclude on `map`** (set in finalize) hides this whole rig from the main render. But the minimap render pass uses an OVERRIDE view-layer where `map` IS visible.

## Notable code patterns
- **The minimap rig is its own mini-scene** — its own cameras + lights, plus the .001 collections (altar.001, behindTheScene.001, oakTrees.001, birchTrees.001, cherryTrees.001, vehicle.001) that are also EXCLUDED from main but visible to the map cameras.
- **Light scale = 245.91** — extreme scale on Area light to make it act like a directional sun across the whole world. Scale on AREA lights changes their physical size, affecting falloff.
- **Both cameras at the same XYZ** (0, 0, 55.74) — they're identical viewing setups; the runtime swaps which is active.
- Sun direction encoded in `rotation_euler = (-0.668, 0.004, -4.378)` — Bruno's specific sunset/sunrise angle (-0.668 rad ≈ -38° elevation, -4.378 rad ≈ -250.8° azimuth, modulo 2π = ~109°).
