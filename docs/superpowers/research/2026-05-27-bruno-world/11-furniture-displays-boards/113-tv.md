## Technique / recipe

**Retro CRT TV with screen + 2-piece compound collider:**

1. **Body mesh** (`Cube`/`Cube`) — the visible TV casing, smoothed at 46° threshold. This is the main visual.
2. **Screen mesh** (`refScreen`/`Cube.009`) — placed 0.12m forward (player-facing). The runtime probably renders the TV screen content (animated effects?) onto this mesh's emissive material.
3. **Back/casing block** (`Cube.005`/`Cube.012`) — co-located with `Cube` (same XY+z), likely the rear cabinet or stand visible from behind/below.
4. **2-cuboid compound collider:**
   - `cuboid.199`: body at z=0.998, scale (1.93, 1.0, 2.01) — the upper TV housing
   - `cuboid.198`: lower at z=0.612, scale (1.93, 1.0, 1.18), offset +0.35m forward in y — the **base/stand** (Bruno's TV sits on a small stand jutting forward)
5. **PhysicalDynamic anchor** at the BODY position (-54.51, 69.15, 1.155) — actual world coords. The TV is pushable.

**Compound collider** = same trick as previous batches: split the collision shape into multiple boxes for a more accurate hitbox. Here it's a 2-piece shape: tall back (housing) + short lower-front (stand).

**`refScreen` named mesh** — runtime reads this by name and attaches the screen-content shader (likely a TSL animated shader showing static, noise, or recorded content).

**Different colliders for actual world position** — `cuboid.199` and `cuboid.198` are at the TV's real coords (-54.51, 69.x, ...), not at a staging area. Compare to other props which stage their colliders. The TV uses the simpler "collider at real position" pattern.

**`Smooth by Angle` (base variant) Input_1=0.802 rad ≈46°** — similar threshold to PlayStation (111, 49° on its body). Bruno picks similar smoothing for similar retro-tech aesthetics.

## Notable code patterns

- **`Cube` as mesh name** (literally `Cube`, no suffix) — Bruno keeps Blender's default primitive name. No renaming.
- **`refScreen` ref-prefix** — runtime hooks the screen-shader to this object by name.
- **Compound collider** (body + base) — 2 cuboids form a T-shape collision profile, accurately matching the TV's silhouette.
- **Forward-offset screen** (Δy=+0.12m from body) — the screen mesh juts slightly out of the body so it's visible without z-fighting and so the emissive material can be authored as a separate shader.
- **All 3 mesh objects at z=1.001** — they're at the same height (mid-TV). Compound is in Y axis (body, screen, casing forward/back), not Z.
- **No staging position** — the colliders at the actual world coords. Different from the template-stack pattern used elsewhere.
- **Pairs with `playstation` (111)** — both retro props live together in the timeMachine zone, 2-3m apart. Same material palette + similar smoothing variants.
- **`refTvPhysicalDynamic`** has both `ref` prefix AND `PhysicalDynamic` suffix — Bruno's runtime reads by name for BOTH the screen-shader hook AND the physics body. One empty does double duty.
